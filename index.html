Why Pipethon

Built different,
by design.

Pipeline composition

Every program is a series of clean transformations. Data flows stage to stage — no tangled callbacks, no deeply nested conditionals.

Pattern matching

Match by type, shape, value, or structure. Tame unpredictable AI output, route branches cleanly, and validate data with elegance.

AI-native primitives

Pipethon is designed from the ground up to integrate with AI models — call, classify, and branch on model output in a single pipeline.

Syntax

Readable by design.

query_handler.pipe 
# Define a pipeline that processes user queries
pipeline handle_query(query: String) -> Response:
  query
    |> sanitize
    |> embed
    |> ai_classify(model="fast")
    |> match:
        Intent.Question(confidence > 0.8) -> answer_pipeline
        Intent.Task                          -> task_pipeline
        Intent.Ambiguous                     -> clarify("Could you rephrase?")
        _                                    -> fallback

# Sub-pipeline for answering questions
pipeline answer_pipeline(intent: Intent):
  intent
    |> retrieve_context
    |> ai_generate(model="deep", temperature=0.4)
    |> format_response
